   Anti-Spoofing

   Amplification Angriff
      Bei einem Amplification Angriff nutzt ein Angreifer den Umstand, dass mit einer kleinen Anfrage eine grosse Antwort ausgelöst werden kann. Die Antwort dieses System lässt sich zudem durch Manipulation der Anfrage an ein anderes System umleiten.

   ARP
      Das Address Resolution Protocol (ARP) ist ein Netzwerkprotokoll, das zu einer IP-Adresse die MAC-Adressen ermittelt und diese Zuordnung gegebenenfalls in den ARP-Tabellen der beteiligten Rechner hinterlegt. Es wird fast ausschließlich im Zusammenhang mit IPv4-Adressierung  verwendet, obwohl es nicht darauf beschränkt ist. Für IPv6 wird diese Funktionalität nicht von ARP, sondern durch das Neighbor Discovery Protocol (NDP) bereitgestellt.

   Passive DNS
      Im Gegensatz zum traditionellen DNS, bei dem ein DNS server angefragt wird um die IP adresse eines hosts zu erhalten, funktioniert passive DNS umgekehrt.

      Es existiert also eine DNS Datenbank, die die DNS Einträge, IP lookups und generelle Statistiken zu einer bestimmten Domain speichert. So kann man auf grund aktiver DNS Kommunikation eine Datenbank mit passiven DNS Daten anlegen. 

      Durch das abspeichern der Daten wird eine spätere Analyse der DNS Information möglich. Man kann somit erkennen was über die Zeit geändert hat. Also zum Beispiel dass die IP von www.example.com von 127.0.0.1 auf 127.0.0.2 geändert hat.

      Mit klassischem DNS ist dies nicht möglich (see `passive DNS <https://securitytrails.com/blog/passive-dns>`_).

   SMAP
      Supervisor Mode Access Prevention (SMAP) is a feature of some CPU implementations such as the Intel Broadwell microarchitecture that allows supervisor mode programs to optionally set user-space memory mappings so that access to those mappings from supervisor mode will cause a trap. This makes it harder for malicious programs to "trick" the kernel into using instructions or data from a user-space program.[1][2] (see `SMAP <https://en.wikipedia.org/wiki/Supervisor_Mode_Access_Prevention>`_)

   SMEP
      So, SMEP `Supervisor Mode Execution Prevention`, as the name states, helps Kernel to prevent the execution of non-privileged code or command in Kernel space. Supervisor Mode Access Prevention (:term:`SMAP`) is designed to complement Supervisor Mode Execution Prevention (SMEP), which was introduced earlier. SMEP can be used to prevent supervisor mode from unintentionally executing user-space code. :term:`SMAP` extends this protection to reads and writes.[2] (see `SMEP <https://en.wikipedia.org/wiki/Supervisor_Mode_Access_Prevention#History>`_)

   Smurf Attacke
      The Smurf attack is a distributed denial-of-service attack in which large numbers of Internet Control Message Protocol (ICMP) packets with the intended victim's spoofed source IP are broadcast to a computer network using an IP broadcast address. Most devices on a network will, by default, respond to this by sending a reply to the source IP address. If the number of machines on the network that receive and respond to these packets is very large, the victim's computer will be flooded with traffic. This can slow down the victim's computer to the point where it becomes impossible to work on. `Wiki Smurf <https://en.wikipedia.org/wiki/Smurf_attack>`_

Router, Server, Switches, Drucker, Computer usw.) von einer zentralen Station aus überwachen und steuern zu können. Das Protokoll regelt dabei die Kommunikation zwischen den überwachten Geräten und der Überwachungsstation. `Wiki SNMP <https://de.wikipedia.org/wiki/Simple_Network_Management_Protocol>`_

   XSRF
      xsrf führt den Bestellprozess beim Opfer automatisch mittels Images-Tags Aufrufe von einer anderen Webseite auf Webshop aus.

   XSS
      xss (deutsch Webseitenübergreifendes Skripting) bezeichnet das Ausnutzen einer Computersicherheitslücke in Webanwendungen. Sie erlaubt es einem Angreifer Script code auf einer Webseite zu platzieren, der dann im Browser eines anderen Benutzers ausgeführt wird, sobald dieser auf die Seite zugreift.
